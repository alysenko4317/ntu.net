Запустити MSVS Installer.
Перевірити, що встановлені .NET Development компоненти та WCF у їх складі.

Створити рішення, яке складатиметься з 3 проєктів:
  - бібліотека класів
  - сервер
  - клієнт

Одразу створюємо ці три проєкти на базі .NET Framework:
  - бібліотека класів
  - сервер — консольний застосунок
  - клієнт — Windows Forms застосунок

Починаємо з бібліотеки класів:
  - додати посилання на `System.ServiceModel`
  - бібліотека міститиме клас, доступний для виклику з клієнтів; цей клас
    розміщується на сервері, і для нього створюється екземпляр
    (оскільки сервер один, доречно використати патерн Singleton)
  - створити клас `ChatService`, який реалізує інтерфейс `IChatService`
  - передбачити три методи: `Connect`, `Disconnect`, `SendMessage`

  - для підключення функціоналу WCF використовуємо атрибути:
        `[ServiceBehavior(InstanceContextMode = InstanceContextMode.Single)]` — для класу
        `[ServiceContract]` — для інтерфейсу
        `[OperationContract]` — для методів

  - сервер зберігатиме перелік користувачів онлайн, наприклад:
      `List<ChatUser> _connectedUsers = new List<>()`
    і відповідно клас `ChatUser` з такими властивостями:
         `Id`
         `Name`
         `OperationContext Context` (зв'язок з користувачем)

  - потрібен двосторонній зв'язок, бо сервер має розсилати повідомлення всім клієнтам,
    тому додаємо інтерфейс `IChatServiceCallback`
  - тут також використовуються атрибути `[ServiceContract]` та `[OperationContract]`
    для методу `SendMessageToClient`
  - цей інтерфейс прив'язується до `ServiceContract` інтерфейсу `IChatService`:
      `[ServiceContract(CallbackContract = typeof(IChatServiceCallback))]`

=======================================

Далі переходимо до реалізації сервера:

      - додаємо новий елемент `WCF Service` (переважно для генерації конфігурації)
          згенерований код можна видалити, бо сервіс реалізовано в бібліотеці
      - у коді потрібно створити host і забезпечити його роботу,
          об'єкт створюється так:
             `using (var host = new ServiceHost(typeof(ChatService)))`
             `{ ... }`
      - додати залежність `System.ServiceModel` (для `ServiceHost`)
      - додати залежність від нашої бібліотеки (для `ChatService`)
      - відредагувати конфіг:
           `behaviourName` встановити в `mex` (metadata exchange)
           `serviceMetadata` — дозволити передавання метаданих про методи
           `serviceDebug` — встановити `true` (можна продемонструвати помилку,
              наприклад, зробити два методи з однаковими іменами, наприклад `SendMessage`)
           у `service` додати `behaviourConfiguration=mex` (див. `behaviourName`)
           `endpoint` — точка підключення
              налаштувати порти, за потреби скоротити адреси
      - після цього можна запускати сервер (можуть знадобитися права адміністратора)

==========================================

Тепер із запущеним сервером пишемо клієнт:

     - додаємо `Add Service Reference`
         вказуємо адресу з протоколом, наприклад `http://localhost:port`
         за потреби змінюємо ім'я на `ChatServiceReference`
     - можна продемонструвати в браузері відкриття цього URL та генерацію помилки (див. `serviceDebug`)
     - також можна показати метадані

     - конфіг клієнта буде згенеровано автоматично
     - створюємо головну форму з кнопками (`connect`, `send`), полем для `nickName`
       і полем для повідомлення, а також виводом `chatHistory`

     - створити клас для взаємодії з сервером — `ChatServerConnector`
         це буде Singleton, бо з'єднання із сервером одне,
         тому робимо приватний конструктор і метод `GetInstance`

     - для взаємодії з сервером використовуємо клас `ChatServiceClient`,
       який згенеровано автоматично; потрібно створити об'єкт цього класу,
       а щоб сервер міг взаємодіяти з клієнтом, `ChatServerConnector`
       повинен реалізовувати інтерфейс `IChatServiceCallback`

     - реалізувати методи `Connect` і `SendMessageToServer`, які викликаються з форми
       (їх можна зробити статичними, оскільки використовується Singleton)

     - щоб заповнювалася `chatHistory`, реалізувати `SendMessageToClient`,
       який звертається до форми:
           `(Application.OpenForms["Form1"] as Form1).lstChatHistory.Items.Add(msg);`
       `lstChatHistory` має бути з модифікатором `public`

     - далі можна запускати кілька клієнтів; якщо в конфігу замінити `localhost`
       на реальну IP-адресу, застосунок працюватиме в локальній мережі

     - потім можна вдосконалювати функціонал (приватні повідомлення тощо)
